# Plane Self-Hosted Deployment

Self-hosted [Plane](https://plane.so) (Community Edition) with Docker Compose, Caddy reverse proxy for automatic HTTPS, SMTP email, and automated PostgreSQL backups.

## Prerequisites

- **Server:** Linux VPS with 4 GB RAM minimum (Ubuntu 22.04 LTS recommended)
- **Docker:** Engine 24+ with Compose v2
- **Domain:** DNS A record pointing to your VPS IP (e.g. `plane.yourdomain.com`)
- **Ports:** 80 and 443 open on the firewall

## Quick Start

```bash
# Clone the repo
git clone https://github.com/andreidrSB/plane-selfhost.git
cd plane-selfhost

# Run the installer (installs Docker if needed, prompts for domain + SMTP)
chmod +x install.sh
./install.sh
```

The installer will:
1. Check/install Docker
2. Prompt for your domain and SMTP settings
3. Generate secure passwords and secrets
4. Pull all Docker images
5. Run database migrations
6. Start all services

## First Run

1. Open `https://your-domain.com` (allow 60-90 seconds for startup)
2. Register — the **first user becomes instance admin**
3. Go to **Settings > General** and set the instance name
4. Go to **Settings > Email** and verify SMTP with a test email
5. Create a **Workspace**, then create **Projects** for each team/company
6. Invite team members via email
7. Use the **Board** tab for Kanban view

## Architecture

```
Internet → Caddy (auto TLS) → Plane Nginx Proxy → Plane Services
                                                    ├── plane-web (Next.js)
                                                    ├── plane-api (Django)
                                                    ├── plane-worker (Celery)
                                                    ├── plane-beat (scheduler)
                                                    └── plane-live (realtime)
Infrastructure: PostgreSQL 15, Valkey 7, RabbitMQ 3.13, MinIO
```

## Backup & Restore

**Automated backups** run daily at 02:00 UTC via a cron container. Backups are retained for 14 days (configurable via `BACKUP_RETENTION_DAYS` in `.env`).

```bash
# Manual backup
docker compose exec plane-backup /usr/local/bin/backup.sh

# Copy backups to host
docker run --rm \
  -v plane-selfhost_plane-backups:/backups \
  -v $(pwd)/local-backups:/out \
  alpine sh -c "cp /backups/*.sql.gz /out/"

# Restore from backup
./scripts/restore.sh ./local-backups/plane_db_20250314_020000.sql.gz
```

## Upgrade

```bash
./scripts/upgrade.sh
```

This pulls the latest Plane images, stops app services (infrastructure stays running), runs migrations, and restarts app services.

## Useful Commands

```bash
# View all logs
docker compose logs -f

# View specific service logs
docker compose logs -f plane-api

# Check service health
docker compose ps

# Restart a single service
docker compose restart plane-api

# Stop everything
docker compose down

# Stop and remove volumes (DESTRUCTIVE — deletes all data)
docker compose down -v
```

## Configuration

All configuration is in `.env` (generated by `install.sh`). See `.env.example` for all available variables.

Key settings:
- `PLANE_DOMAIN` — your domain name
- `EMAIL_*` — SMTP configuration for notifications
- `BACKUP_RETENTION_DAYS` — how long to keep backups (default: 14)
- `LISTEN_HTTP_PORT` — internal proxy port (default: 8080)
